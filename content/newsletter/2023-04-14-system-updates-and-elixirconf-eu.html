Official project updates

 

property_table, v0.2.3

Fix compiler warnings with Elixir 1.15

nerves, v1.10.2

Change BuildRunners.Docker to use GitHub Container Registry by default

nerves_livebook, v0.9.0

  • Update Livebook to v0.9.1
  • Update Nerves systems to nerves_system_br 1.22.5 versions. This includes Erlang 25.3 and fixes to the Raspberry Pi 3 to support TensorFlow Lite. (TFLite isn’t built into Nerves Livebook yet)
  • Update all dependencies to their latest versions.
  • Fixed LEDs on SRHub images (Thanks to Eric Oestrich)

circuits_quickstart, v0.5.6

  • Update Nerves systems to nerves_system_br v1.22.5 (Erlang/OTP 25.3, Buildroot 2022.11.3)
  • Update all dependencies to latest

qmi, v0.8.5

  • Fixed incorrect enum value for serving system registration state reports. This could cause crashes if the value 3 (registration denied) was received.
  • Fixed spelling of :fw_update_discontinuous_frame

nerves_system_br, v1.22.5

This is a security/bug fix release for v1.22.4.

We switched from Docker Hub to the GitHub Container Registry for new official system images. Existing Docker Hub images are not going away. However, new ones will be at ghcr.io/nerves-project/nerves_system_br. This change should not affect anyone unless you manually fetch the images.

  • Package updates
    • Buildroot 2022.11.3
    • Fwup 1.10.0

    Supporting the Nerves project

    We want to encourage everyone to contribute in whatever way works for them. Here are some ways we currently recommend:

    • Write or port a new hardware library and include it in the Elixir Circuits collection.
    • Give money to the project via Open Collective.
    • Get in touch about taking over maintenance duties for some libraries, we might be able to provide you hardware.
    • Apply for an EEF stipend on something Nerves-related and build it. We can help if you have questions about this.
    • Send corrections or improvements for documentation wherever it fails to help you.
    • Write about Nerves, give talks about Nerves. Make videos about Nerves. It is all good.

    Finally, if you have questions about the newsletter or want to suggest something you can simply respond to this email it will go directly to me, Lars, who edits this thing. Let me know what you think.

    - Lars, Underjord.io
