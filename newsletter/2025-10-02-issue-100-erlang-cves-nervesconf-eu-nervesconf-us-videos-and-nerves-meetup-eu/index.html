NervesConf EU feat. Goatmire Elixir

NervesConf EU is in the bag. As the organizer I did a write-up on NervesConf EU & Goatmire Elixir .

First day of the 3 days under the Goatmire umbrella was NervesConf EU. Because Nerves comes first in my book ;)

The write-up has much more detailed thoughts but to summarize it was very successful. I don't have exact isolated numbers for NervesConf from the door but I believe we had about 120 of the 140 ticket holders in attendance at least.

Full photo gallery is up . I'll share some Nerves-related ones below.

And YES. We plan to PUBLISH RECORDED TALKS. No timeline but next few months is the idea.

Also happy to report the conference did not run a deficit. The workload was significant and we took out no compensation but the financials of the event were sound enough.

Importantly to the Nerves-interested we handed out the Wisteria name badges created by Gus Workman ( Protolux Electronics ) who put in an absolutely hilarious amount of work. Based on his Trellis board ( Nerves system ). Firmware is here . Lots of learnings for how to do this better at an event next time but it mostly worked and people had fun with them. That's roughly 180 eInk Nerves devices distributed to the population of developers.

Amazing talks on Nerves were given. I'll not rehash the speaker list. Look it up on the website. Enjoy the pictures :)

circuits_gpio, v2.1.3

IMPORTANT: Please upgrade to this version or newer since it fixes an issue where interrupts stopped getting sent. This bug was triggered when a GenServer owning an input GPIO with interrupts enabled was restarted. It could be triggered via manual closing and reopening of a GPIO as well.

  • Bug fixes
    • Fix an issue where the Erlang pid that should have received interrupts on GPIO changes was lost.

      • Bug fixes
        • Fix hardcoded RELEASE_DIR setting so that it’s automatically computed from the release’s base directory. This allows non-Nerves use cases to change the base directory. Thanks to @ziopio for finding and fixing the issue.

      circuits_quickstart, v0.13.2

      • Updates
        • Update Nerves systems to nerves_system_br v1.31.7 (Erlang/OTP 27.3.4.3, Buildroot 2025.02.6)

      ring_logger, v0.11.4

      • Changes
        • Persist logs on terminate if appropriate (Thanks @taun-fellowes)
        • Support date entry navigation in the log viewer (Thanks @Shivaji-Dhepale, @Suraj Borate)
        • Fix deprecation warnings in example code (Thanks @axelson)
        • Fix Elixir 1.19 warnings
        • Drop support for Elixir <1.13

      nerves_hub_link, v2.9.0-rc.1

      • Added
        • The way firmware updates are downloaded and installed has been improved to support recovery of downloads in the event of application crashes and device reboots. The default update strategy has been encapsulated in a new module called NervesHubLink.UpdateManager.StreamingUpdater, while the new optional caching updater strategy is implemented in NervesHubLink.UpdateManager.CachingUpdater. New update strategies can be added by implementing the NervesHubLink.UpdateManager.Updater behaviour. Please read the module documentation for more information.

      nerves_hub_link, v2.8.1

      • Added

        • Send the firmwares validation status to NervesHub when the connection is established via an optional Client.firmware_validated?/0 callback. A default implementation is provided, while supporting overriding with custom logic.
        • Poll for firmware validation changes and send an update to NervesHub when the firmware is validated.
        • Detect if a firmware revert has occurred and send this information to NervesHub when the connection is established. This is supported via an optional Client.firmware_auto_revert_detected?/0 callback. A default implementation is provided, while supporting overriding with custom logic.
        • Custom Clients can now be implemented using use NervesHubLink.Client, eliminating the need to implement all Client behavior callbacks.

      • Updated

        • Support for Elixir 1.13.x has been removed

      How you can help Nerves

      Contribute in the way that works for you:

      • Send corrections or improvements for documentation wherever it fails to help you.
      • Write about Nerves, give talks about Nerves. Make videos about Nerves. It is all good.
      • Write or port a new hardware library and include it in the Elixir Circuits collection .
      • Get in touch about taking over maintenance duties for some libraries, we might be able to provide you hardware.
      • Apply for an EEF stipend on something Nerves-related and build it. We can help if you have questions about this.

      Finally, if you have questions about the newsletter or want to suggest something you can simply respond to this email.
