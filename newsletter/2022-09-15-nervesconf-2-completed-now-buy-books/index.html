Official project updates

 

circuits_uart, v1.5.1

Don’t even try using the RS485 ioctls if not set by the user. This fixes an issue on Allwinner devices, but probably others, where getting RS485 settings works, but setting them, even if unchanged always fails. Thanks to @jjcarstens for the fix.

nerves_ssh, v0.4.1

Default SSHSubsystemFwup config would overwrite any user defined config preventing FWUP handling customization (thanks @ConnorRigby!)

nerves_bootstrap, v1.11.2

Remove references to :build_embedded since that option is planned for removal in Elixir 1.15 and it wasn’t needed.

nerves_system_br, v1.20.6

This release reverts an update to libp11 in v1.20.5 that currently breaks the OpenSSL engine integration with ATECC cryptographic chips (like NervesKeys). It has no other changes.

nerves_system_br, v1.20.5

This is a security/bug fix release for v1.20.4.

  • Package updates
    • Buildroot 2022.05.2
    • Also see Buildroot 2022.05.1 changes
    • Erlang/OTP 25.0.4

nerves, v1.9.1

This is a patch release that fixes trivial tooling issues found when using Elixir 1.14 and Erlang 25.0.4. It’s expected to be a safe update from v1.9.0.

Updates on NervesConf II and III(!)

From Todd Resudek:

We held NervesConf II on Aug 31, the morning before ElixirConf IRL started.

Due to hardware limitations, we sold out the conference, which was limited to 50 people. SmartRent supplied a project kit with a SmartRent hub, cellular modem, and sim card.

Next year's NervesConf will be expanded to accommodate 100 people and will be more of a traditional conference including a CFP for speakers. Be on the lookout in the Spring of 2023 for that CFP to open.

Very excited to see that it sold out and that it keeps going.

You can check out this Twitter thread for some shoutouts and follow that account to find out when the next one is brewing.

Scenic 0.11 released

The Scenic framework has a good synergy with Nerves and the new version is a major step (a breaking one unfortunately) with lots of changes to support the future plans and some of the very interesting work with Kry10 OS and all that. The driver picture seems a fair bit improved. Read more here.

Supporting the Nerves project

We want to encourage everyone to contribute in whatever way works for them. Here are some ways we currently recommend:

  • Write or port a new hardware library and include it in the Elixir Circuits collection.
  • Give money to the project via Open Collective.
  • Get in touch about taking over maintenance duties for some libraries, we might be able to provide you hardware.
  • Apply for an EEF stipend on something Nerves-related and build it. We can help if you have questions about this.
  • Send corrections or improvements for documentation wherever it fails to help you.
  • Write about Nerves, give talks about Nerves. Make videos about Nerves. It is all good.

Finally, if you have questions about the newsletter or want to suggest something you can simply respond to this email it will go directly to me, Lars, who edits this thing. Let me know what you think.

- Lars, Underjord.io
